\section{Simulation}

\subsection{Testbench}
\subsection*{1. Simulation Configuration and Parameters}
The testbench configures the Device Under Test (DUT) with specific hardware and timing constants to match the intended operational environment:
\begin{itemize}
    \item \textbf{System Clock}: The simulation is set to a frequency of 5 kHz. This corresponds to a clock period of 200,000 ns.
    \item \textbf{Hardware Parameters}: The system is instantiated with 16 LEDs, 12 clockwise steps (\texttt{NumCW}), and 8 anti-clockwise steps (\texttt{NumACW}).
    \item \textbf{Temporal Resolution}: A \texttt{TimeStep} parameter of 500 ms is applied to define the duration of each LED state transition.
    \item \textbf{Cycle Definitions}: The testbench calculates \texttt{RUN20\_CYCLES} and \texttt{RUN5\_CYCLES} based on the clock frequency to simulate specific time intervals.
\end{itemize}

\subsection*{2. Clock Generation and Initialization}
\begin{itemize}
    \item \textbf{Clock Stimulus}: A perpetual clock signal is generated by toggling the \texttt{clk} register at half-period intervals (100,000 ns).
    \item \textbf{System Reset}: Upon initialization, the \texttt{reset} signal is driven high and held for 10 clock cycles to ensure all internal registers reach a known state.
    \item \textbf{Initial Conditions}: The repeat signal (\texttt{rep}) is initially set to 0 to prevent the FSM from advancing before the reset is released.
\end{itemize}

\subsection*{3. Test Stimulus Phases}
The simulation executes two distinct phases to verify the controller's reactivity to user input:
\begin{itemize}
    \item \textbf{Phase 1 (Active Operation)}: The \texttt{rep} signal is set to 1. The simulation runs for the duration of \texttt{RUN20\_CYCLES} (representing approximately 40 seconds of system time) to observe the continuous repetition of the clockwise and anti-clockwise LED sequence.
    \item \textbf{Phase 2 (Sequence Termination)}: The \texttt{rep} signal is cleared to 0. The simulation continues for \texttt{RUN5\_CYCLES} (representing 5 seconds) to verify that the system completes its current cycle and transitions to the \texttt{OFF\_STATE} correctly before the simulation finishes.
\end{itemize}

\subsection*{4. Output Monitoring}
The testbench utilizes the \texttt{\$monitor} task to provide real-time logging of the simulation time, the 16-bit \texttt{LedOut} bus, the \texttt{rep} status, and the \texttt{reset} state. This ensures that any deviations in the output pattern can be identified relative to the control signals.

\subsection{Simulation results}

The simulation results shown in the waveform verify that the \texttt{Ring\_Flasher\_EX} design operates correctly according to the following specifications:

\begin{itemize}[leftmargin=*, labelsep=0.5cm]
    \item \textbf{Sequence and Direction Accuracy}: 
    The waveform clearly demonstrates the state transitions from \texttt{CW\_STATE} to \texttt{ACW\_STATE}. The active bit in the \texttt{LedOut[15:0]} bus increments for exactly 12 steps (Clockwise) before reversing direction for 8 steps (Anti-clockwise) as defined by the \texttt{NumCW} and \texttt{NumACW} parameters.

    \item \textbf{Timing and Clock Synchronization}: 
    The \texttt{divclk} pulses occur at precise intervals derived from the \texttt{TimeStep} parameter of 500ms. The simulation time axis confirms that the LED transitions are synchronized with these pulses, ensuring the system maintains the required 500ms-per-step pace.

    \item \textbf{Fading Trail Implementation}: 
    The multi-level brightness requirement is met through the PWM logic. As seen in the waveform, the \texttt{LedOut} bits exhibit varying duty cycles; the current position is at 100\% (Level 5), while the preceding bits show a decreasing duty cycle (Level 4, 3, 2, 1), confirming the mathematical decrement of the brightness level.

    \item \textbf{Control and Repeat Logic}: 
    The simulation validates the response to the \texttt{rep} signal. When \texttt{rep} is high, the FSM successfully loops through the CW and ACW sequences. Upon setting \texttt{rep} to 0, the system correctly completes its current cycle and transitions to the \texttt{OFF\_STATE}, darkening all LEDs once the fading process is finalized.

    \item \textbf{Reset Integrity}: 
    At the start of the simulation, the \texttt{reset} signal successfully initializes all registers to 0, ensuring no unintended LED activity occurs until the controller leaves the \texttt{IDLE\_STATE}.
\end{itemize}

\textbf{Conclusion}: Based on the visual evidence from the waveform and the recorded logs, the design successfully meets all functional and timing requirements specified for the Ring Flasher system.
