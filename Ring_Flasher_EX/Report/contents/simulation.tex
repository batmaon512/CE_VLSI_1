\section{Simulation}

\subsection{Testbench Architecture}

The professional testbench is organized into independent task-based test cases, each with its own reset sequence to ensure test isolation and reproducibility. The testbench is structured as follows:

\begin{itemize}
    \item \textbf{System Clock}: 5 kHz (CLK\_PERIOD = 200,000 ns)
    \item \textbf{LED Configuration}: 16 LEDs with 12 clockwise steps and 8 anti-clockwise steps
    \item \textbf{Timing}: TimeStep parameter set to 500 ms per state transition
    \item \textbf{Task-Based Design}: Each test case is implemented as an independent task that applies its own reset before executing its test scenario
\end{itemize}

\subsection{Test Cases and Functional Verification}

\subsubsection{TC1: Normal Operation (Legacy Baseline)}

\textbf{Objective}: Verify normal ring flasher operation with continuous repetition followed by controlled fade-out.

\textbf{Test Flow}:
\begin{enumerate}
    \item Apply reset: hold \texttt{reset = 1} for 10 clock cycles
    \item Release reset: set \texttt{reset = 0} and wait 100 cycles for system settlement
    \item Activate sequence: set \texttt{rep = 1} and run for $40 \times \text{CLK\_FREQ}$ cycles ($\approx 8$ seconds at 5 kHz)
    \item Deactivate sequence: set \texttt{rep = 0} and run for $5 \times \text{CLK\_FREQ}$ cycles ($\approx 1$ second) to observe fade-out
\end{enumerate}

\textbf{Expected Behavior}:
\begin{itemize}
    \item While \texttt{rep = 1}: CW motion for exactly 12 steps with LED incrementing positions, then ACW motion for exactly 8 steps with LED decrementing positions
    \item Pattern repeats continuously while \texttt{rep = 1}
    \item Upon \texttt{rep = 0}: FSM completes the current CW or ACW cycle before entering OFF\_STATE (no immediate stop)
    \item After current cycle completes: LEDs gradually fade as brightness levels decrement on each \texttt{divclk} pulse
    \item When all LED brightness levels reach zero: \texttt{off\_all} signal triggers and FSM transitions back to IDLE\_STATE
    \item All control signals synchronized to \texttt{divclk} at 500 ms intervals
\end{itemize}

\subsubsection{TC2: Short Burst Test}

\textbf{Objective}: Verify system response to brief activation pulses and proper cycle completion before fade-out.

\textbf{Test Flow}:
\begin{enumerate}
    \item Apply reset
    \item Set \texttt{rep = 1} and run for $2 \times \text{CLK\_FREQ}$ cycles ($\approx 0.4$ seconds)
    \item Set \texttt{rep = 0} and run for $2 \times \text{CLK\_FREQ}$ cycles to observe fade-out
\end{enumerate}

\textbf{Expected Behavior}:
\begin{itemize}
    \item FSM transitions to CW\_STATE and advances LED position for a few steps
    \item Upon \texttt{rep = 0}: FSM completes the current CW phase before entering OFF\_STATE
    \item After cycle completion: brightness decays on each \texttt{divclk} pulse until all LEDs fade
    \item Validates correct handling of incomplete sequence cycles with proper termination
\end{itemize}

\subsubsection{TC3: Repeat Toggle Test}

\textbf{Objective}: Verify FSM robustness when \texttt{rep} signal is toggled during operation.

\textbf{Test Flow}:
\begin{enumerate}
    \item Apply reset
    \item Set \texttt{rep = 1} for $3 \times \text{CLK\_FREQ}$ cycles
    \item Set \texttt{rep = 0} for $1 \times \text{CLK\_FREQ}$ cycle
    \item Set \texttt{rep = 1} again for $3 \times \text{CLK\_FREQ}$ cycles
    \item Set \texttt{rep = 0} for $1 \times \text{CLK\_FREQ}$ cycle
\end{enumerate}

\textbf{Expected Behavior}:
\begin{itemize}
    \item First activation: LED sequence starts from position 0, advances during CW phase
    \item First deactivation: FSM completes current cycle, enters OFF\_STATE, LEDs fade to zero
    \item \texttt{off\_all} triggers when all LEDs are dark, FSM transitions to IDLE\_STATE
    \item Second activation: FSM returns to CW\_STATE and resumes new sequence cycle
    \item Demonstrates correct state recovery and continuous sequence operation
    \item Validates \texttt{off\_all} feedback mechanism during complete fade-out-to-idle transitions
\end{itemize}

\subsubsection{TC4: Reset Re-entry Test}

\textbf{Objective}: Verify system initialization and recovery when reset is asserted during active operation.

\textbf{Test Flow}:
\begin{enumerate}
    \item Apply reset
    \item Set \texttt{rep = 1} for $2 \times \text{CLK\_FREQ}$ cycles
    \item Mid-run reset: assert \texttt{reset = 1} for 3 clock cycles, then release
    \item Resume operation: set \texttt{rep = 1} for $3 \times \text{CLK\_FREQ}$ cycles
    \item Deactivate: set \texttt{rep = 0} for $1 \times \text{CLK\_FREQ}$ cycle
\end{enumerate}

\textbf{Expected Behavior}:
\begin{itemize}
    \item Upon reset assertion, all brightness levels immediately return to zero
    \item All LED outputs go low regardless of PWM state
    \item After reset release, FSM returns to IDLE\_STATE
    \item Re-application of \texttt{rep = 1} correctly restarts sequence from position 0
    \item System resumes normal operation with proper cycle completion before OFF\_STATE
    \item Validates reset integrity and synchronous design principles
\end{itemize}

\subsubsection{TC5: Long Stability Test}

\textbf{Objective}: Verify sustained operation over extended duration without timing drift or register overflow.

\textbf{Test Flow}:
\begin{enumerate}
    \item Apply reset
    \item Set \texttt{rep = 1} for $60 \times \text{CLK\_FREQ}$ cycles ($\approx 12$ seconds)
    \item Set \texttt{rep = 0} for $5 \times \text{CLK\_FREQ}$ cycles
\end{enumerate}

\textbf{Expected Behavior}:
\begin{itemize}
    \item CW/ACW sequences repeat 5+ complete cycles with consistent timing
    \item Each cycle executes exactly 12 CW steps followed by 8 ACW steps
    \item No jitter or phase shift in \texttt{divclk} pulse generation
    \item No register overflow or saturation effects
    \item Upon \texttt{rep = 0}: system completes final cycle then enters OFF\_STATE
    \item Brightness decay pattern remains linear and predictable
    \item Validates long-term clock divider stability and counter arithmetic
\end{itemize}

\subsubsection{TC6: Fade-out and off\_all Transition Test}

\textbf{Objective}: Verify the end-of-sequence detection mechanism and correct transition to IDLE\_STATE.

\textbf{Test Flow}:
\begin{enumerate}
    \item Apply reset
    \item Set \texttt{rep = 1} for $10 \times \text{CLK\_FREQ}$ cycles
    \item Set \texttt{rep = 0} and observe for $10 \times \text{CLK\_FREQ}$ cycles
    \item Monitor \texttt{off\_all} signal during fade-out
\end{enumerate}

\textbf{Expected Behavior}:
\begin{itemize}
    \item Upon \texttt{rep = 0}: FSM completes current CW or ACW cycle before entering OFF\_STATE
    \item In OFF\_STATE: LED brightness levels decrement on each \texttt{divclk} pulse
    \item Brightness decrements from level 5 down to level 0 sequentially
    \item When all LED brightness levels reach zero, \texttt{off\_all} signal pulses high
    \item FSM transitions back to IDLE\_STATE upon \texttt{off\_all = 1}
    \item Validates NOR reduction gate operation and proper feedback control loop closure
\end{itemize}

\subsection{Simulation Results Summary}

The comprehensive testbench validates the following functional areas:

\begin{itemize}[leftmargin=*, labelsep=0.5cm]
    \item \textbf{State Machine Correctness}: All four states (IDLE, CW, ACW, OFF) transition according to FSM specification and input conditions.
    
    \item \textbf{Sequence Timing}: CW and ACW sequences execute for exactly 12 and 8 steps respectively, with transitions synchronized to \texttt{divclk}.
    
    \item \textbf{Brightness Control}: PWM duty cycles correctly implement 6 brightness levels (0--5) with proper decay during fade-out.
    
    \item \textbf{Reset Functionality}: Synchronous reset properly clears all state and returns the system to a clean initial condition.
    
    \item \textbf{Feedback Control}: The \texttt{off\_all} signal correctly detects when all LEDs have faded and triggers FSM state transition.
    
    \item \textbf{Clock Divider Stability}: Divider maintains consistent timing across extended operations without drift or overflow.
\end{itemize}

All test cases pass and demonstrate that the Ring Flasher design meets its functional and timing specifications.
